Скрипт выполняет анализ access-логов сервера nginx в указанной в конфиге папке, формирует отчет ф html-форммате со 
статистикой по каждому адресу в логе, который помещает в папку отчетов (тоже задается в конфиге). Обрабатывает как plain
так и упакованные gzip файлы логов.

При запуске формирует отчет по самому последнему из ранее не обработанных логов. Обрабатывался или нет лог определяется по
наличию соответсвующего файла отчета.

Для запуска отчета со стандартными параметрами, перейти в папку со скриптом и выполнить команду
python log_analyzer.py

Для запуска отчета со своими параметрами, перейти в папку со скриптом и выполнить команду
python log_analyzer.py --config <[путь/]имя_файла_конифига>

Конфиг - файл формата json, содержащий следующие ключи (указаны параметры по-умолчанию):

    "REPORT_SIZE": 1000               - максимальное кол-во строк в итоговом отчете
    "REPORT_DIR": "./reports"         - папка, куда складывать готовые отчеты
    "LOG_DIR": "./log"                - папка, где искать логи для анализа
    "TS_DIR": tempfile.gettempdir()   - папка, в которую будет записан файл со временем окончания работы скрипта (log_analyzer.ts)
    "LOGGING_DIR": None               - папка для расположения лога работы скрипта, если None - вывод будет производиться в stdout
    "MAX_PARSE_ERRORS": 0.6           - максимально допустимый порог ошибок разбора строка лога (кол-во ошибок/кол-во строк лога),
                                        по превышении отчет сформирован не будет.

В своем конфиге можно опустить любые ключи, тогда будут использоваться параметры по-умолчанию.

Сформированный отчет представляет собой таблицу с полями:
url         - адрес
count       - сколько раз строка с этим адресом встречается в логе
count_prc   - % соотношение кол-ва для этой строки к общему кол-ву строк лога
time_avg    - среднее время обработки запроса к url
time_max    - максимальное зафиксированное время обработки запроса
time_med    - медиана по времени выполнения запроса
time_perc   - % соотношение суммарного времени выполнения запроса к общему суммарному времени по всем запросам
time_sum    - суммарное время выполнения запросов с таким url

Данные в отчете упорядочены по убыванию суммарного времени обработки url (time_sum). В отчет выведено 1-ые config["REPORT_SIZE"] строк
с наибольшим суммарным временем обработки url.
